## Comparison: Perplexity Brain/Memory vs Onyx Brain/Memory

### ✅ **What Onyx DOES Have (and it's actually quite good)**

Onyx already has the **"Dream" / session backfill pattern** that Perplexity uses! Looking at `backend/onyx/background/celery/tasks/brain/tasks.py`:

1. **Scheduled Session Backfill** — The `brain_self_improvement()` Celery task runs daily and:
   - Collects recent chat sessions (last 14 days or since last run)
   - Extracts durable facts, entities, concepts, workstreams, and notes
   - Creates/updates memory "pages" automatically
   - Links related memories together (graph edges)
   - Cites the specific sessions (and optionally documents) that created each memory

2. **Memory Categories** — Same as Perplexity: `entities`, `concepts`, `workstreams`, `notes`

3. **Source Attribution** — Memories track which sessions (and documents) they came from

4. **User-Controlled Settings** — `brain_enabled`, `brain_use_connectors`, `brain_focus_instructions`

### 🔴 **What Perplexity Has That Onyx is Missing**

| Feature | Perplexity | Onyx |
|---------|------------|------|
| **Real-time inline memory creation** | Memories form *during* conversation (not just nightly) | Only nightly batch via `brain_self_improvement` |
| **"Use search history" toggle** | Past searches searchable as recall sources | Not implemented |
| **Space-scoped memories** | Same API with `collection_uuid` for workspace isolation | Memories are user-global only |
| **Custom "Dream" instructions UI** | `/rest/sse/computer/memory/dream-instructions` endpoint with save/load | Only `brain_focus_instructions` on User model |

| **Trigger on-demand Dream run** | `POST /rest/sse/computer/memory/trigger-dream` | Only scheduled (no manual trigger endpoint) |
| **Wiki-style pages with history** | Revisions, diff history, manual editing | Memories are flat text, no revision history |

---

### 🟡 **Key Architecture Difference**

**Perplexity's "Dream" flow:**
```
User chats → Memories formed inline (light extraction) 
          ↓
Nightly "Dream" run → Organizes into wiki pages, links, categories
          ↓
Dream creates/revises wiki pages with full revision history
```

**Onyx's current flow:**
```
User chats → (nothing immediate)
          ↓
Nightly `brain_self_improvement` → Creates/updates Memory rows
          ↓
No revision history, no wiki structure
```

### Bottom Line

**Onyx already has the core "Dream" / session-backfill pattern** — the nightly `brain_self_improvement` task does exactly what Perplexity's Dream does. The gaps are mostly:
